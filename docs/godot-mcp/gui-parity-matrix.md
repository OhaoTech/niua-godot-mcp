# Godot MCP GUI Parity Matrix

The NIUA Godot MCP aims for practical parity with the visible Godot editor GUI: what you can do by hand in the editor, an agent can do through direct tools or tested lower-level primitives. This matrix maps each Godot GUI area to its current MCP coverage.

Status keys:

- `read-only`: available through the current bridge/MCP.
- `partial-write`: current bridge/MCP can modify a limited safe subset.
- `practical-parity`: the agent workflow is covered by direct tools or tested lower-level primitives; remaining exact GUI chrome is documented as a Godot public-API limitation.
- `planned`: required for GUI parity, not implemented yet.
- `unsupported`: no implementation exists yet.
